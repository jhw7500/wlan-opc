> 기준 사양서: `docs/opc_vhl_protocol_Rev1.00_KO.md` (無線基板共通制御通信仕様書 Rev1.00 / 2026-05-25, 落合庸央)

> 목적: 사용자 약식 문의 18항목을 **사양 §절 번호**로 정렬하고, 각 항목의 (전제 정확성 / 사내 결정 현황 / 발주처 문의 가치)를 판정해 발주처(CONTEC / 落合) 발송용으로 정리.

**판정 범례**
- 🟢 유효 — 그대로 발주처에 물을 가치 있음
- 🟡 확인용 — 사내 이미 확정·결정, "우리 해석이 맞는지" 확인 톤으로 격하
- 🔵 설계 제안 — 사양 변경/개선 제안

- 🔴 중복 — 사양에 이미 명시되어 발주처 문의 불필요

## 3. 발주처 발송용 — 주제 그룹 정리

> 18항목 + 누락 10건을 **7개 주제 + 누락**으로 재편. 🔴6번 제거(또는 ①에 흡수), 🟡2·11·13은 "확인" 톤.

### ① 에러코드 정의 체계 (상)
- **§3.3.1·§3.3.5·§3.3.6** 무효문자 정의 — password(0x0011/0x0013) + ESSID(0x0015) 모두 (4번)
- **§3.3.2·§3.3.9** Login/Logout/Indication 의 0x0001·0x0002·0x0013 의미 정합표 (5+6번 통합)
- **§3.3.6** IP리스트 비정상 boundary(start 없는 continue/end) 응답코드 — 0x0018 제안 확인 (7번)
- **§3.3.8** SetRadio **apply 런타임 실패** 전용 코드 — 0x0050 제안 확인 (16번)

### ② SetRadio 의미론 (상)
- **§3.3.8** 무선설정 OK 기준: 설정 기록 완료 vs 재접속 성공까지 (17번)
- **§3.3.6·§3.3.8** ESSID를 IP리스트가 아닌 무선설정에 두는 제안 + 현재 묶은 의도 질의 (9번)
- **§3.3.4** device-info WLAN FREQ/CH = 설정값(spec) vs 접속(live)값 확정 — 사내 토글 신설, 출하 기본 설정값 (G11)

### ③ 네트워크 토폴로지 (상) — *구성도 한 장으로 일괄 답변*
- **§3.3.4·§3.3.6** 설정 IP가 유선/무선 중 무엇인지 + 투명 브릿지 모델 확인 (10번)
- **§3.3.4·§3.3.6** gateway 불요(브릿지)·GW/NTP=0 허용 확인 (11번 + G6)
- **§3.3.4·§3.3.6** NTP 서버 위치·cross-subnet 라우팅 범위 (12번)
- **§2/전반** 실제 라인 구성도(각 장치 IP/mask/GW + AP/컨트롤러) (18번)

### ④ IP리스트 시퀀스 (중)
- **§3.3.6** start=0x0000/continue=0x0001/end=0x0002 채택값 확인 + 문서 p22/p24 상충 정정 요청 (7+13번)

### ⑤ 장애검출통지 (중)
- **§3.4.5** CPU/Memory/Disk/Network 임계·단위(%/Mbps)·지속·재통지 기준 + Memory(0x0002) 미발행(swapless) 동의 여부 (14번 + G)

### ⑥ 상태·동작 (중)
- **§3.3.7·§3.3.1** timeout 자동로그아웃 시 ChangeIp 미적용·폐기 해석 확인 (8번)
- **§3.3.3·§3.4.1** "기동 중" 상태 지속시간·VHL 대기방식 (15번)

### ⑦ Capability·Dual (중)
- **§3.3.4** r/k/v: 장치지원 vs 링크협상 활성, silicon 실광고 subset (3번 + G5)
- **§2.2·§3.4** Dual Station 납품 범위 필수 여부 + wlan_id 식별필드 부재 (1번 + G4)

### ⑧ 누락 신규 (상)
- **§3.1.2** 기본 UDP 포트 확정 (G1)
- **§3.2.1** Protocol Version 협상 (G2)
- **§3.4.6** ResetNotice 자율리셋 트리거/cause (G3)
