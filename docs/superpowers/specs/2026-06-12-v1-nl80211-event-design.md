# V1 설계 — nl80211 이벤트 → 잔여 indication 3종 연결

- **작성일:** 2026-06-12
- **근거:** spec-conformance D7 잔여(WlanStatusChange/Roaming/ApDisconnect 미발행), V1
- **선행 결정:** wlan_id 잠정 정책 — 주 WLAN(mlan0) 기준 발행 (2026-06-12 사용자 결정, 이슈 #35 항목 6)

## 현황 (배관은 완비, producer만 부재)

| 구성요소 | 상태 |
|---|---|
| Indication emitter 3종 (`opcd/indication.c::opcd_ind_{wlan_status,roaming,ap_disconnect}`) | 구현·테스트 완료 |
| 이벤트 타입/union (`opcd/platform.h::OPCD_PEVT_*`, idx 필드 포함) | 정의 완료 |
| Consumer (`opcd/opcd.c::on_platform_event` + epoll 등록) | 구현 완료 |
| **Producer** (`platform_nxp.c::nxp_event_fd / nxp_drain_events`) | **no-op — 본 설계 대상** |

## 제약 (확정)

- **의존성 금지**: libnl 미사용 — raw `AF_NETLINK`/`NETLINK_GENERIC` 소켓 + 수동 genl/nla
  파싱 (mlanutl 직접 ioctl과 같은 결).
- **정책/사실 분리**: producer(nxp)는 ifindex→idx를 **사실대로** 보고하고, mlan0-only
  잠정 정책은 consumer(`on_platform_event`)가 적용한다 — 고객사 답변으로 정책이 바뀌어도
  producer 불변.
- **coalesce 책임**: platform.h 계약 "implementations must coalesce / debounce internally —
  opcd will not throttle".
- 사양 §3.4 페이로드: WlanStatusChange(status+CH), Roaming(SNR/RSSI/MAC/CH),
  ApDisconnect(reason msg id/result code/MAC).

## 설계

### 1. 소켓 수명 (nxp_init / nxp_event_fd)

- `nxp_init()`에서: genl `CTRL_CMD_GETFAMILY("nl80211")`로 family id + `mlme` multicast
  group id 해석(bounded, 실패 시 event_fd=-1로 degrade — 현행과 동일 무이벤트 운용),
  `NETLINK_GENERIC` 소켓 생성 → `mlme` 그룹 `setsockopt(NETLINK_ADD_MEMBERSHIP)` →
  `O_NONBLOCK|SOCK_CLOEXEC`. `if_nametoindex("mlan0"/"mlan1")` 캐시.
- `nxp_event_fd()` = 그 소켓 fd. opcd가 epoll에 등록(기존 코드 그대로).

### 2. 이벤트 매핑 (nxp_drain_events)

비차단 recv 루프에서 nl80211 메시지를 파싱해 콜백으로 전달:

| nl80211 | OPCD_PEVT | 비고 |
|---|---|---|
| `NL80211_CMD_CONNECT` (status==0) | `WLAN_STATUS(ASSOCIATED, ch)` | CH는 `NL80211_ATTR_WIPHY_FREQ`→CH 변환 |
| `NL80211_CMD_DISCONNECT` | `AP_DISCONNECT(reason, mac)` + `WLAN_STATUS(UP)` | reason=`NL80211_ATTR_REASON_CODE`, mac=`NL80211_ATTR_MAC` |
| `NL80211_CMD_ROAM` | `ROAMING(mac, ch, snr, rssi)` | SNR/RSSI는 §3 참조 |
| `NL80211_CMD_CH_SWITCH_NOTIFY` | `WLAN_STATUS(CHANNEL_CHANGE, ch)` | DFS/CSA |

- `NL80211_ATTR_IFINDEX` → 캐시된 ifindex로 idx(0/1) 결정, 미지 ifindex는 무시.
- coalesce: 한 drain 사이클 내 동일 (kind, idx)는 마지막 이벤트만 전달 (작은 고정 배열).
- recv 오류(ENOBUFS 포함)는 1회 로그 후 0 반환 — ENOBUFS는 이벤트 유실이므로
  WLAN_STATUS 재동기(현재 링크 상태 1회 재보고)로 보정.

### 3. SNR/RSSI (Roaming 페이로드)

nl80211 ROAM 이벤트에는 SNR/RSSI가 없다. 1차: 기존 `get_link()` 경로(link.json)의
캐시 값을 사용(이미 snr/rssi 보유, `opcd_platform_link_t`). 캐시 미존재 시 0 보고.
`NL80211_CMD_GET_STATION` 동기 질의는 후속 검토(드레인 콜백 안 블로킹 회피).

### 4. 파서 분리 (테스트 가능성)

genl/nla 파싱을 순수 함수로 분리: `opcd/nl80211_parse.{c,h}` —
`int nl80211_parse_evt(const uint8_t *msg, size_t len, int family_id, opcd_nl_evt_t *out)`.
소켓/ifindex 의존 없음 → 실기 캡처 바이트로 호스트 단위 테스트(`test_nl80211_parse.c`).
`platform_nxp.c`는 소켓 I/O + idx 매핑 + coalesce만 담당.

## 테스트

- 호스트: `test_nl80211_parse` — CONNECT/DISCONNECT/ROAM/CH_SWITCH 캡처 프레임 4종 +
  malformed(짧은 nla, 미지 cmd) 거부. stub 빌드(`make check`)에 nxp 코드 미포함 — 파서만 링크.
- 실기(NXP88W9098): manual-runthrough §indication — AP 절단/재접속/로밍 유도 후
  vhlctl 수신 확인. ENOBUFS 보정 경로는 이벤트 폭주(연속 스캔)로 확인.

## 비범위

- mlan1(DUAL) 이벤트의 indication 라우팅 — 고객사 답변(#35 항목 6) 후. producer는 idx를
  이미 보고하므로 consumer 정책 1줄 변경으로 대응.
- T9 watchdog/fault 자율리셋 producer — 동일 배관(`OPCD_PEVT_RESET_NOTICE`) 재사용, 별도 작업.
- FaultDetect — 폴링 프로브로 기구현(T6 INTERIM).

## 검증 기준
